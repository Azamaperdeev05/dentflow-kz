# 2. Жүйе тұжырымдамасы (System Concept)

## 2.1 Жоба идеясы

DentFlow KZ — стоматологиялық клиниканың барлық негізгі процестерін біріктіретін біртұтас цифрлық экожүйе. Жүйе «бір терезе» принципімен жұмыс істейді: пациент, дәрігер және клиника әкімшісі бір платформадан өз міндеттерін атқарады.

## 2.2 Мәселе анықтамасы

Қазіргі стоматологиялық клиникалардағы негізгі мәселелер:

| № | Мәселе | Салдары |
|---|--------|--------|
| 1 | Қабылдауға телефон арқылы жазылу | Уақыт жоғалтады, қателер болады |
| 2 | Қағаз тіркеу жүйесі | Деректер жоғалады, іздеу қиын |
| 3 | Емдеу тарихының жоқтығы | Дәрігер ақпарат алмайды |
| 4 | Қаржылық есептің қиындығы | Ай сайынғы статистика қолмен жасалады |
| 5 | Деректер қауіпсіздігі | Медициналық ақпарат қорғалмайды |

## 2.3 Шешім стратегиясы

DentFlow KZ келесі стратегиямен шешім ұсынады:

```
Мәселе → Шешім → Нәтиже

Телефон жазылу → Онлайн күнтізбе → 24/7 жазылу
Қағаз тіркеу → Цифрлық DB → Деректер қауіпсіз
Тарих жоқ → Медициналық карта → Толық тарих
Қаржы қиын → Автоматты есеп → Бір басу статистика
Қауіпсіздік жоқ → 2FA + шифрлау → HIPAA деңгейі
```

## 2.4 Мақсатты аудитория

### Негізгі пайдаланушылар

| Рөл | Адам саны (орташа клиника) | Негізгі міндеттер |
|-----|---------------------------|-------------------|
| **Пациент** | 500–2000 | Жазылу, тарих көру, чат |
| **Дәрігер** | 3–15 | Қабылдау, емдеу, төлем |
| **Админ** | 1–3 | Бақылау, бекіту, аудит |

### Қосымша пайдаланушылар (болашақ)
- Ресепшн
- Бухгалтер
- Клиника меңгерушісі

## 2.6 Жүйенің ерекшеліктері

1. **Толық қазақ тілі** — UI, error messages, validations барлығы қазақша
2. **Windows 11 дизайн** — Fluent Design System иконографиясы
3. **Көпқабатты қауіпсіздік** — 2FA, шифрлау, аудит, rate limiting
4. **Шифрланған чат** — AES-256-CBC деңгейінде хабарлама қорғау
5. **Модульдік архитектура** — Әр рөлдің жеке layout, nav, компоненттері
